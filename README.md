# Sistema de Gerenciamento de Pedidos

Sistema desenvolvido em **Java** utilizando **JDBC** para persistência de dados em banco relacional, aplicando conceitos de **Programação Orientada a Objetos (POO)** e arquitetura em camadas.

O projeto simula um sistema de gerenciamento de vendas, permitindo o cadastro e gerenciamento de **produtos**, **clientes** e **pedidos**, além da manipulação de um **carrinho de compras**, onde é possível adicionar e remover itens e finalizar uma compra com persistência no banco de dados.

## ✨ Funcionalidades

* Cadastro, consulta, atualização e remoção (CRUD) de produtos
* Cadastro completo de clientes
* CRUD de pedidos
* Carrinho de compras com adição e remoção de produtos
* Finalização de pedidos com gravação no banco de dados
* Interface de linha de comando (CLI) baseada em menus
* Persistência utilizando JDBC

## 🛠️ Tecnologias

* Java
* JDBC
* SQL
* Banco de Dados Relacional
* Git
* GitHub

## 📂 Estrutura do Projeto

O sistema foi organizado seguindo princípios da Programação Orientada a Objetos, separando responsabilidades entre entidades, camada de acesso a dados (DAO), regras de negócio e interface via console, tornando o código mais modular, reutilizável e de fácil manutenção.

## 🎯 Objetivo

Este projeto foi desenvolvido com o objetivo de consolidar conhecimentos em Java, Programação Orientada a Objetos e integração com banco de dados utilizando JDBC, implementando operações CRUD e um fluxo completo de gerenciamento de pedidos sem a utilização de frameworks, proporcionando uma compreensão mais aprofundada do funcionamento da camada de persistência.
